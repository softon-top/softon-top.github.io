从软路由到 Hugo：我的家庭服务器搭建全记录

Saturday, January 31, 2026
1 minute read

我搭建个人站点的初衷，不是为了追求流量，只是想拥有一个完全属于自己的“数字小窝”，记录技术探索、日常思考，也顺便折腾一下各类软路由、虚拟机相关的玩法。初衷落地的第一步，就是搞定硬件设备和基础环境，毕竟“巧妇难为无米之炊”，今天就和大家详细拆解，我这套个人建站方案用到的所有设备、具体成本，以及各个组件的用途，全程干货无废话，适合想入门自建站点、喜欢折腾软路由的朋友参考～ 先交代一个基础前提：我家的网络是电信宽带，速率日常够用，用来支撑个人站点的本地搭建、远程调试和数据推送完全没问题，这也是整个建站方案的网络基础，无需额外为网络单独增加成本（除了日常缴纳的宽带费）。

一、核心设备清单：软路由为主，配件为辅

不同于传统的硬路由，我选择了软路由作为整个方案的核心硬件——软路由的灵活性更高，可扩展性极强，既能当主路由管理家庭网络，又能搭建虚拟机、容器，承载建站相关的服务，一举多得。下面是所有设备的详细清单，包含型号、配置、购买渠道和成本（价格为我购买时的实际花费，仅供参考，不同渠道、不同时段可能有波动）。

1. 核心硬件：康耐信软路由（裸机）

软路由是整个系统的“心脏”，所有的虚拟机、容器、网络管理都依赖它运行，所以我在选择时优先考虑了性能足够、稳定性强的型号，最终在咸鱼入手了康耐信软路由裸机（裸机意味着不含内存、硬盘，需要自己额外搭配，灵活性更高）。

核心配置：CPU 为 Intel N8505，这款CPU属于低功耗、高性能的入门级工控CPU，日常运行软路由系统、多个虚拟机和容器完全无压力，不会出现卡顿、死机的情况，而且功耗低，长期开机也不用担心电费过高。

购买渠道及成本：咸鱼，900元（裸机价，不含内存、硬盘）。

2. 配套配件：内存、硬盘

软路由裸机没有内置内存和硬盘，需要自己搭配，考虑到要运行PVE系统、多个虚拟机（VM）和容器（LXC），还要存储站点相关文件，所以在配件上没有太节省，选择了够用且稳定的配置，分别在不同渠道入手了内存和硬盘。

  • 内存：16G DDR4 笔记本内存 × 2 根，总内存32G。之所以选择32G内存，是因为虚拟机和容器运行时会占用一定的内存，多开几个（比如我目前的2个VM+2个LXC）也不会出现内存不足的情况，后续如果想增加更多服务，也有扩展空间。购买渠道及成本：咸鱼，单根200元，2根共计400元。

  • 硬盘：1T NVMe SSD 固态硬盘。选择SSD而非机械硬盘，主要是因为SSD的读写速度更快，运行PVE系统、虚拟机和站点服务时，响应会更流畅，而且SSD的稳定性更好，适合长期开机运行，存储站点源码、配置文件等也足够用。购买渠道及成本：拼多多，400元。

3. 总成本核算

以上所有硬件的总成本的为：软路由裸机（900元）+ 内存（400元）+ 硬盘（400元）= 1700元。

补充说明：这个成本是核心硬件的花费，不包含电信宽带费（日常正常缴纳）、GitHub Pages（免费），也不包含后期可能新增的配件，整体属于入门级偏高一点的配置，主要是为了稳定性和扩展性，如果只是单纯建站、不折腾太多虚拟机，也可以选择更低配的内存和硬盘，能节省几百元成本。

二、基础环境搭建：PVE系统 + VM/LXC 容器

硬件组装完成后，第一步就是安装系统——我没有直接安装软路由系统或建站系统，而是先安装了PVE（Proxmox Virtual Environment），这是一款开源的虚拟化平台，简单来说，它可以让我们在一台软路由上，同时运行多个独立的系统，也就是虚拟机（VM）和容器（LXC），相当于把一台物理机“分成”了多台虚拟的电脑，每台虚拟电脑可以运行不同的服务，互不干扰。

1. 先搞懂：VM（虚拟机）和 LXC（容器）的区别

很多朋友第一次接触虚拟化，会分不清VM和LXC的区别，我用最通俗的话给大家解释，不用纠结复杂的技术术语，看懂核心区别就好：

  • VM（虚拟机）：相当于“完整的虚拟电脑”。它会模拟一整套电脑硬件（CPU、内存、硬盘、网卡等），然后在上面安装完整的操作系统（比如Windows、Linux、软路由系统），每个VM都是独立的，拥有自己的内核，互不影响。优点是兼容性强，能运行几乎所有操作系统和软件；缺点是占用资源较多（因为要模拟完整硬件、运行完整系统）。

  • LXC（容器）：相当于“轻量化的虚拟环境”。它不会模拟完整的硬件，而是共享宿主系统（这里就是PVE）的内核，只隔离应用程序的运行环境，相当于在同一台电脑上，开辟了多个独立的“小空间”，每个空间运行不同的服务。优点是占用资源少、启动快，适合运行简单的服务；缺点是兼容性不如VM，只能运行和宿主内核兼容的系统（主要是Linux相关）。

对我来说，两者搭配使用最合适：需要运行完整系统（比如软路由系统、桌面系统）的场景，用VM；需要运行简单服务（比如文件管理、监控）的场景，用LXC，既节省资源，又能满足不同的需求。目前我搭建的环境中，主要用了2个VM，LXC暂时只创建了备用，后续会根据需求增加服务。

2. 两个核心VM：软路由 + 建站主机

在PVE上创建了两个虚拟机，各自承担不同的角色，相辅相成，构成了我个人建站的基础环境，下面分别介绍：

（1）VM1：ImmortalWrt 主路由系统

第一个虚拟机，我安装了 ImmortalWrt 系统（一款开源的软路由系统，基于OpenWrt开发，稳定性更强，功能更丰富），作为整个家庭网络的主路由，负责管理网络连接、分配IP、防火墙设置等。

其中最关键的一个配置，就是在 ImmortalWrt 里安装了 OpenClash——熟悉软路由的朋友应该都知道，OpenClash 是一款常用的代理工具，主要作用是解决一些海外资源访问的问题（比如后续我可能会用到一些海外的建站插件、调试工具），同时也能优化网络连接速度。

至于具体的网络结构、OpenClash 配置方法，以及 ImmortalWrt 系统的其他设置，内容比较多，也比较细致，后续我会单独写一篇文章详细拆解，感兴趣的朋友可以持续关注～

（2）VM2：Zorin 系统 + Hugo 建站

第二个虚拟机，是我真正的“建站主机”，安装了 Zorin 系统——一款基于Ubuntu开发的Linux桌面系统，界面简洁、易用，对新手很友好，而且稳定性强，适合长期运行建站服务。

在 Zorin 系统上，我安装了 Hugo 建站工具——Hugo 是一款静态站点生成器，最大的优点是速度快、轻量化，不需要搭建复杂的数据库，只需要编写Markdown文档，就能快速生成静态站点（也就是我现在的个人blog）。

我的建站流程很简单：在本地（Zorin 系统里）用 Hugo 编写文章、调试站点样式，完成后，通过Git工具将站点源码推送到 GitHub Pages，GitHub Pages 会自动部署站点，生成可公开访问的网址，这样一来，既不需要购买服务器，也能实现站点的公开访问，而且完全免费、稳定。

三、总结与后续规划

到这里，我个人建站的硬件设备、成本明细，以及基础的环境搭建，就全部介绍完了。整体来说，这套方案的核心是“软路由+虚拟化”，既满足了家庭网络的需求，又实现了本地建站、免费部署，而且有足够的扩展性，后续可以根据自己的需求，增加更多的服务（比如文件服务器、监控系统等）。

后续的文章，我会逐步拆解更细致的内容：比如 PVE 系统的安装与配置、ImmortalWrt 软路由的详细设置（含 OpenClash 配置）、Hugo 建站的具体操作（从安装到推送 GitHub Pages），以及整个网络结构的详细拆解。

如果大家有疑问，比如设备选型、成本优化、环境搭建中的坑，都可以在评论区留言，我会尽自己所能分享经验～ 毕竟，折腾的过程，本身就是一种乐趣呀。

对于使用Hugo建站的博主而言，截图插入（比如教程类文章、网站效果展示）是高频需求——手动截图、保存本地、上传图床、复制链接、粘贴到文章，整个流程繁琐且耗时。今天就给大家带来一套高效解决方案：利用 ShareX（免费截图工具）+ Cloudflare（免费CDN+存储），实现「截图即上传、上传即获链接」，全程自动化，彻底解放双手，让你专注于文章创作本身。

本教程分为三大部分，从域名准备、图床搭建到工具配置，每一步都附带详细操作截图（文字清晰可落地），新手也能轻松跟着走，无需专业技术基础。

第一部分：通过Porkbun注册域名，并绑定到Cloudflare管理

搭建稳定且高可用的图床，自定义域名是绕不开的核心前提 —— 它将替代 Cloudflare 提供的临时域名，成为图片资源的永久唯一标识。

将域名托管至 Cloudflare 后，我们不仅能借助其免费 CDN 大幅提升图片的加载速度，还能一站式获得专业的 DNS 解析与安全防护服务。作为建站唯一的成本投入，域名的重要性无需赘述（详见前文：Hugo+Github Pages+Cloudflare 免费图床 个人低成本建站保姆级教程）。若你计划长期运营个人站点，强烈建议注册专属域名并以此搭建图床。本站亲测采用「Porkbun 注册域名 + Cloudflare 免费托管」的组合，全程流程简洁高效，且价相比国内服务商更具优势，非常适合开发者入门。

先重点介绍Porkbun域名注册商的核心优势，这也是我选择它的关键原因，尤其适配Hugo建站用户的需求，对比国内阿里云、腾讯云等注册商，优势十分明显：

  • 价格更实惠：核心优势之一，同等后缀的域名（如.com、.top、.xyz等），Porkbun首年注册价普遍比国内注册商低30%-50%，且无隐藏消费，续费价格也相对稳定，适合个人博主长期使用，用于图床搭建性价比拉满；

  • 支持支付宝直付：无需绑定信用卡、PayPal，直接支持支付宝付款，流程和国内购物付款一样便捷，解决了很多用户国际付款不便的痛点，付款后即时生效，无需等待审核；

  • 无需备案，全程无门槛：Porkbun注册的域名属于国际域名，无需进行国内ICP备案，注册完成后可直接使用，省去了国内注册商域名备案的繁琐流程（备案通常需要3-7天，还需准备主体证明材料），大大缩短了图床搭建周期；

  • 口碑优质，服务稳定：Porkbun是ICANN认证的域名注册商，连续2023-2025年被《今日美国》评为全球最佳域名注册商，Trustpilot上拥有大量5星好评，支持全年365天客服响应，且不会强制推销无关服务，使用体验简洁流畅。

1.1 注册域名（Porkbun平台，全程支付宝直付）

  2. 在搜索框输入想要的域名（可以先只输入前缀，比如softxxx），点击搜索，会批量查询哪些域名是否可注册；

    点击「sort by price」会按价格排序，可以看到最低费用，首年注册低至1.54美元，但是要注意下面的续费费用，有的首年很便宜，但是续费价格较高（避坑）。

    目前能找到的还是.top域名，首年1.63美元，续费4.63美元，算是性价比很高的了。

  3. 选择可注册的域名，点击「+」加入购物车，点击购物车图标，结算时选择注册年限（首年最优，后续可续费），确认域名信息无误后，点击「Checkout」进入付款页面；

    在这里可以选择注册年限，如果想长期用建议直接选3年或5年甚至直接10年，比注册1年再每年续费还要优惠。

    确认后，在右下角点击「Continue to Billing」进入付款页面；

  4. 在该页面直接选择Alipay（支付宝），点击「Continue to Alipay」，会直接跳到支付宝界面进行支付。

  5. 付款完成后，返回Porkbun官网，进入「Domains」控制台，即可看到已注册的域名，记住其「Manage」（管理）入口，后续需要修改DNS服务器，完成与Cloudflare的绑定。

小贴士：域名后缀优先选择 .com、.top、.xyz 等常见后缀，Porkbun这类后缀的首年价格通常在1-5美元（折合人民币几元到几十元），比国内同类型域名便宜不少；由于Porkbun属于国际注册商，注册的域名无需备案，绑定Cloudflare国际节点后，国内外均可快速访问，完全适配图床使用需求。

1.2 将Porkbun注册的域名，绑定到Cloudflare完成托管

Cloudflare提供免费的DNS解析、CDN加速服务，将Porkbun注册的域名托管到这里后，后续我们搭建的图床可以借助其全球节点，让图片在国内外都能快速加载，同时避免图片被墙、加载缓慢的问题。操作流程和国内注册商域名托管一致，重点是修改Porkbun域名的DNS服务器。

  1. 注册/登录Cloudflare账号：访问Cloudflare官网（https://dash.cloudflare.com/sign-up），输入邮箱、密码，点击「Sign up（创建账户）」，然后验证邮箱即可完成注册（全程免费，无需绑定信用卡）；

  2. 添加域名到Cloudflare：登录后，点击页面顶部「+Add」，在下拉项目选择「Connect a domain」；

    然后输入你需要托管的域名，直接选推荐的「Quick scan for DNS records」，下面的AI爬虫策略选默认设置即可，然后点击「Continue」继续；

  3. 选择Cloudflare计划：弹出套餐选择页面，我们选择「Free（免费计划）」，点击「Continue（继续）」，免费计划完全满足个人图床需求，包含无限流量、DNS解析、基础CDN加速等核心功能；

  4. 验证并导入DNS记录：Cloudflare会自动扫描你域名当前的DNS记录（如果是新注册的域名，可能没有任何记录，直接点击「Continue」即可）；扫描完成后，无需修改任何记录，直接点击「Continue」；

  5. 修改域名DNS服务器：这是关键一步！Cloudflare会给出两组DNS服务器地址（coleman.ns.cloudflare.com 和 ziggy.ns.cloudflare.com），我们需要回到Porkbun域名控制台，将域名的DNS服务器修改为这两组地址；

  6. Porkbun域名控制台：登录Porkbun，进入「Domains」，找到对应域名，点击「NS（域名服务器）」，删除默认的Porkbun DNS服务器，粘贴Cloudflare提供的两组DNS地址，点击「Submit」保存并确认；

  7. 注意：DNS服务器修改后，生效需要时间（通常5-30分钟，最长不超过24小时），耐心等待即可，可通过 https://www.whatsmydns.net/#NS/ 输入域名，查询DNS是否生效。

  8. 确认托管成功：回到Cloudflare页面，点击左侧导航栏「Domain」，进入域名管理页面，如果页面显示域名为「Status：Active」，说明Porkbun注册的域名已经成功托管到Cloudflare，后续我们就可以在Cloudflare上管理这个域名的解析、CDN等功能了。

补充说明：Porkbun修改DNS服务器的操作比国内部分注册商更简洁，无需人工审核，保存后即时提交修改请求；如果后续你更换其他国际注册商，修改DNS服务器的步骤类似，找到域名的「Nameservers」设置，替换为Cloudflare提供的DNS地址即可。

第二部分：利用Cloudflare Storage建立免费图床

Cloudflare Storage（R2 Object Storage）是Cloudflare推出的对象存储服务，专门用于存储图片、文件等静态资源，核心优势是「免费额度高、出口流量全免」——提供10GB免费存储（足够个人博客存储上万张截图）、每月100万次上传、1000万次读取，且出口流量完全免费，无需担心后续产生额外费用，是个人图床的最优选择之一。

这一步我们将完成「创建存储桶→配置公开访问→获取API凭证」，为后续ShareX自动上传做好准备，全程在Cloudflare后台操作，无需编写代码。

2.1 开通Cloudflare R2存储服务

  1. 登录Cloudflare后台，在左侧导航栏找到「Storage & databases」→「R2 object storage」，点击「Overview」进入；

  2. 首次使用会提示「开通R2服务」，点击「开通」，此时会要求绑定支付方式（信用卡或PayPal），这里无需担心，仅用于身份验证，不会自动扣费，免费额度内完全免费；

    1. 支持绑定：信用卡（Visa、MasterCard、银联均可）、PayPal；

      我使用的银联的信用卡，非常方便。

    2. 绑定完成后，再次点击「开通R2服务」，几秒钟即可开通成功。

2.2 创建存储桶（用于存放截图图片）

存储桶相当于我们图床的「文件夹」，所有通过ShareX上传的截图，都会自动存放在这个存储桶中，我们需要创建一个专属存储桶，并配置相关参数。

  1. 开通R2后，点击页面右上角「Create bucket」，进入创建页面；

  2. 填写存储桶信息：

    1. Bucket name（存储桶）名称：自定义，建议简洁且有辨识度（比如 softxxx-img、hugo-img-bed等），注意名称是全局唯一的，如果提示重名，更换一个即可；

    2. Location（位置）选择：有小技巧可参考——亚太地区（APAC）国内访问更快，但国际访问受限；北美西部（WNAM）配合Cloudflare CDN，全球访问速度更均衡，建议默认：Automatic；

    3. 存储类：默认选择「标准（Standard）」即可，适合频繁访问的截图资源。

  3. 点击「Create bucket」，几秒钟后，存储桶即可创建完成，进入存储桶详情页，后续的配置都在这里进行。

2.3 配置存储桶公开访问（关键步骤）

R2存储桶默认是私有的，也就是说，上传的图片无法通过公网访问，我们需要配置公开访问，让图片能够通过域名直接访问（即图床的核心功能），推荐使用「自定义域名」配置（避免默认临时域名限速）。

  1. 进入存储桶详情页，点击顶部「Settings（设置）」标签，向下滚动找到「Custom Domains（自定义域名）」；

  2. 点击「Add（添加）」，输入我们想要用于图床的子域名（比如 img.softon.top）；

  3. 点击「继续」，Cloudflare会自动在你的域名DNS记录中添加一条CNAME记录，无需手动操作，可以在domain管理里面查看绑定的DNS记录；

  4. 当设置页面Custom Domains显示Access「Enabled」，说明自定义域名已经绑定成功，后续访问图片的URL就是「https://img.softon.top/图片名称.png」；

  5. 小贴士：避免使用 Public Development URL「r2.dev 子域名」（仅适合测试），该临时域名有访问速率限制，博客访问量稍大就会导致图片加载失败，生产环境务必使用自定义域名。

2.4 获取API凭证（用于ShareX关联）

ShareX要实现自动上传图片到Cloudflare R2，需要通过API凭证验证身份，我们需要创建一个API令牌，并保存好关键信息（仅显示一次，务必妥善保管）。

  1. 回到R2对象存储的「Overview（概述）」页面，在右下角的 Account Details点击「Manage（管理）」；

  2. 点击「Create Account API token（创建API令牌）」，填写令牌信息：

    1. 令牌名称：自定义，比如 sharex-upload-token，用于区分用途；

    2. 权限：选择「对象读和写（Object Read & Write）」，确保ShareX能够正常上传和读取图片；

    3. 存储桶：选择我们刚才创建的存储桶（或选择「所有存储桶」）。

  3. 点击「Create Account API token（创建API令牌）」，创建完成后，页面会显示三个关键信息，务必立即复制并保存到记事本（关闭页面后再也无法查看，只能重新创建）：

    1. 访问密钥ID（Access Key ID）；

    2. 访问密钥（Secret Access Key）；

    3. S3 API端点（类似 https://xxxxx.r2.cloudflarestorage.com，可后续在R2概述页右下角Account Details看到。

  4. 至此，Cloudflare R2免费图床已经搭建完成，我们拥有了「自定义域名+免费存储+CDN加速」的图床，接下来配置ShareX，实现截图自动上传。

第三部分：ShareX详细操作（界面、功能、步骤、设置）

ShareX是一款免费、开源、无广告的全能截图工具（仅支持Windows系统），不仅支持多种截图模式，还能实现截图编辑、自动上传、链接复制等功能，在GitHub上收获了超过33.8k星标，深受开发者、博主喜爱。配合我们刚才搭建的Cloudflare图床，就能实现「截图→上传→复制链接」全程自动化，完美适配Hugo文章创作。

这一部分我们从「软件安装→界面功能→图床配置→操作步骤→高级设置」，全方位拆解ShareX的使用方法，确保你能快速上手。

3.1 ShareX下载与安装（免费无广告）

  1. 下载ShareX：访问ShareX官网（https://getsharex.com/），点击首页「Download」，下载最新版本的安装包（Windows系统，约20MB，无需付费）；

  2. 安装ShareX：双击安装包，按照提示下一步即可，无需修改默认设置（可自定义安装路径）；

  3. 启动ShareX：安装完成后，桌面会生成快捷方式，双击启动，启动后会自动最小化到系统托盘（右下角），不会占用桌面空间。

3.2 ShareX界面与核心功能说明

ShareX的界面简洁明了，核心功能集中在「截图、上传、工具」三大模块，新手无需复杂学习，熟悉几个关键按钮即可上手，下面介绍常用界面和功能：

3.2.1 主界面（打开ShareX后显示）

主界面分为4个核心区域，对应不同功能，一目了然：

  1. 截图（左侧第一个按钮）：核心功能区，包含多种截图/录屏模式，常用的有：

    1. 矩形截图：拖动鼠标选择任意矩形区域，截图后自动进入编辑界面；

    2. 窗口截图：自动识别当前打开的窗口，点击即可截取整个窗口（不含桌面背景）；

    3. 全屏截图：截取整个电脑屏幕（所有显示器，可设置单屏）；

    4. 滚动截图：截取网页、文档等可滚动的内容（比如长网页，自动滚动并拼接成一张图片）；

    5. 屏幕录制：录制屏幕视频或GIF动图，适合展示操作流程。

  2. 上传（左侧第二个按钮）：用于手动上传本地图片、文件，也可查看上传历史（所有通过ShareX上传的图片，都能在这里找到）；

  3. 工具（左侧第四个按钮）：有很多实用的工具可以选择，最重要的是图像编辑器，截图完成后，点击此处可进入编辑界面，常用功能有：

    1. 标注工具：添加箭头、矩形、圆形、文字等，用于突出截图重点（适合教程类截图）；

    2. 裁剪工具：裁剪截图多余部分，调整图片尺寸；

    3. 模糊工具：模糊截图中的敏感信息（比如密码、个人信息）；

    4. 格式转换：将截图保存为PNG、JPG、GIF等格式（默认PNG，清晰度最高）。

  4. 设置（左侧中间区域）：核心配置区，程序设置、动作设置、上传目的地设置，后续图床关联、快捷键设置、自动上传设置等，都在这里操作。

3.2.2 系统托盘功能（最小化后）

ShareX启动后最小化到系统托盘，右键点击托盘图标，可快速访问常用功能：

  • 快速截图：选择任意截图模式，无需打开主界面；

  • 上传文件：手动选择本地文件上传到图床；

  • 历史记录：查看截图和上传历史；（这是后期最常用且最方便的功能）

  • 退出：关闭ShareX（退出后无法使用快捷键截图）。

3.3 ShareX核心操作：关联Cloudflare图床（关键步骤）

这是实现「截图自动上传」的核心配置，我们需要在ShareX中添加Cloudflare R2图床，填写之前获取的API凭证，让ShareX能够正常上传图片到我们搭建的图床。

  1. 打开ShareX，点击设置区域的「Destination settings（目的地设置）」；

  2. 在跳出的设置窗口中，选择「Amazon S3」（因为Cloudflare R2兼容S3 API，无需选择其他选项）；

  3. 点击下方「Amazon S3」，进入详细配置页面，填写我们在第二部分获取的API凭证和图床信息，每一项都要填写正确：

    1. 访问密钥ID：粘贴我们保存的「Access Key ID」；

    2. 秘密访问密钥：粘贴我们保存的「Secret Access Key」；

    3. 存储桶名称：填写我们创建的Cloudflare R2存储桶名称（比如 softxxx-img）；

    4. 端点：粘贴我们保存的「S3 API端点」（比如 https://xxxxx.r2.cloudflarestorage.com）；

    5. 区域：默认auto（Cloudflare R2自动适配）；

    6. URL样式：选择「自定义域名」，填写我们绑定的图床自定义域名（比如 https://img.softon.top）；

    7. 上传路径：填写时点击输入框，右边会有输入提示，可以按照自己的喜好设置，我的设置为「%y/%mo/%d」（按照年/月/日上传，将图片存放在存储桶的相应文件夹下，在cloudflare的R2存储桶的总揽页面可以看到相应的文件夹结构和上传的图片文件，可以在这里进行管理）；

    8. 其他选项默认即可，无需修改。

  4. 填写完成后，点击主界面左侧下部的「调试」→「测试图片上传」，如果弹出「上传成功」的提示，说明ShareX已经成功关联Cloudflare图床；在「调试日志」可以看到上传信息，在「历史记录」可以看到上传的时间、文件名、上传成功的网址信息；如果失败，需要检查API凭证、存储桶名称、自定义域名是否填写正确，同样可以在调试信息里面查找错误信息根据信息排查；

  5. 关联成功后，点击「确定」，保存设置，返回主界面。

3.4 ShareX日常操作步骤（截图→自动上传→复制链接）

配置完成后，日常使用非常简单，全程只需2步，即可获取截图的图床链接，直接粘贴到Hugo文章中：

  1. 启动截图：

    1. 方式1：打开ShareX主界面，点击「截图」，选择对应的截图模式（比如矩形截图）；

    2. 方式2：使用快捷键（推荐，效率更高），默认快捷键：矩形截图「Ctrl+PrintScreen」、全屏截图「PrintScreen」、窗口截图「Alt+PrintScreen」、滚动截图「Ctrl+Alt+PrintScreen」；

    3. 截图时，拖动鼠标选择区域（矩形截图），松开鼠标后，自动进入编辑界面（可跳过编辑，直接上传）。

  2. 编辑与上传：

    1. 编辑截图（可选）：在编辑界面，使用标注工具、裁剪工具等，编辑完成后，点击右上角绿色对勾；

    2. 自动上传：编辑完成后，ShareX会自动上传图片到Cloudflare图床（无需手动点击上传），上传过程中会显示进度条；

    3. 复制链接：上传成功后，图片的图床链接会自动复制到剪贴板（右下角会弹出「URL已复制到剪贴板」的提示）；

    4. 粘贴使用：先在历史记录里面选择需要复制的图片，鼠标右键→复制→Markdown图像，然后在Markdown编辑器（比如Obsidian、VS Code），粘贴即可插入截图。

小贴士：如何设置截图后自动打开图像编辑和自动上传：在主界面左侧点击「截图后的任务」，选中「打开图像编辑器」和「上传图片」即可，如果不想自动打开编辑截图，可取消勾选「打开图像编辑器」，截图后会直接自动上传，效率更高。注意：选中后字体变粗，但是与不选中区别不大，可以仔细对比一下。

3.5 ShareX高级设置（优化使用体验，适配Hugo建站）

根据Hugo建站的需求，我们可以对ShareX进行一些优化设置，让截图上传更贴合文章创作习惯，推荐3个常用设置：

3.5.1 自定义快捷键（提升效率）

默认快捷键可能不符合你的使用习惯，可自定义快捷键，比如将矩形截图设置为「Ctrl+Q」，更方便操作：

  1. 进入ShareX主界面→「快捷键设置」，点击「添加」；

  2. 在「任务」下拉菜单中，选择对应的功能（比如「屏幕截图→截图矩形区域」）；

  3. 点击「设置热键」，按下想要设置的快捷键（比如Ctrl+Q），点击「确定」；

  4. 保存设置后，即可使用自定义快捷键启动截图。

3.5.2 设置图片格式与质量（适配Hugo加载速度）

Hugo博客追求加载速度，截图图片不宜过大，可设置图片格式为PNG（清晰度高+体积小），并压缩质量：

  1. 进入ShareX主界面→「动作设置」→「图像」；

  2. 图像格式：选择「PNG」（默认），如需更小体积，可选择「JPG」（质量设置为80%左右，兼顾清晰度和体积）；

  3. PNG位深：默认即可；

  4. 保存设置，后续截图会自动按此格式和质量生成。

3.5.3 设置自动命名（避免图片重名）

如果截图名称重复，上传时会覆盖之前的图片，可设置自动命名规则，让每张截图都有唯一名称：

  1. 进入ShareX主界面→「动作设置」→「上传」；

  2. 默认即可，也可以根据需要自定义；

  3. 保存设置，后续截图会自动按设置的规则命名，避免重名覆盖。

3.5.4 设置截图本地文件夹及路径

默认情况下，ShareX会将截图保存到系统默认的个人文件夹中，路径较深且不易查找。建议将截图保存到自定义本地文件夹（文件夹结构最好和cloudflare的bucket相同），便于后续管理。

  1. 进入ShareX主界面→「应用程序设置」→「路径」；

  2. 勾选「使用自定义截图文件夹」；

  3. 子文件夹模式建议和之前的上传路径一致。

3.5.5 设置导入导出

我们在ShareX中完成了图床关联、快捷键、路径等一系列个性化配置后，通过导入导出功能，可以实现「配置备份」「多设备同步」「快速恢复」，避免更换电脑、重装系统或软件后，重新繁琐配置。

  1. 进入ShareX主界面→「应用程序设置」→「设置」；

  2. 可以选中设置和历史记录，也可以根据需要选择；

  3. 选择导出保存设置，后续或者重装后可以导入设置。

3.5.6 用好历史记录

ShareX的历史记录是非常实用的功能，直观的显示了截图的时间、文件名、图床链接、预览图，可以很方便的在文章中插入截图。在你还没有开始写文章的时候可以先对某些操作和使用场景进行截图保存，避免了需要的时候再去重新还原场景。

OpenClaw 实战：打造你的 AI 私人管家

别让你的 AI 助手只会聊天，让它真正帮你干活！

痛点：你的 AI 管家是不是只会动嘴？

兄弟们，有没有这种经历：

  • 每天打开终端，得先敲一堆命令启动服务
  • 想查个日志，得翻半天目录
  • 飞书 API 额度蹭蹭掉，心在滴血
  • 系统啥时候崩的完全不知道，等发现早就凉了

别慌，今天咱们就解决这些问题。我要带你用 OpenClaw 搭建一个真正的 AI 私人管家——它会自动汇报、自动清理、自动升级，还能用缓存把 API 调用砍掉 99.9%。

直接上干货。

第一步：给飞书插件装上"缓存大脑"

先说最痛的点：飞书 API 额度。

默认情况下，OpenClaw 的飞书插件每分钟都会发起健康检查。算笔账：

1 分钟 1 次 × 60 分钟 × 24 小时 × 30 天 = 43,200 次/月

飞书免费版额度才 50,000 次/月，这就没了？

怎么解？ 给 probe.ts 加上 24 小时内存缓存。

核心代码（直接抄）

文件位置：/usr/lib/node_modules/openclaw/extensions/feishu/src/probe.ts

// 在文件顶部添加缓存常量
const PROBE_CACHE_TTL_MS = 24 * 60 * 60 * 1000; // 24 小时
const probeCache = new Map<string, { result: any; timestamp: number }>();

// 在 probe 函数中，先检查缓存
const cacheKey = `${gatewayUrl}:${port}`;
const cached = probeCache.get(cacheKey);
if (cached && Date.now() - cached.timestamp < PROBE_CACHE_TTL_MS) {
  return cached.result;
}

// 正常发起请求...
const result = await fetch(...);

// 写入缓存
probeCache.set(cacheKey, { result, timestamp: Date.now() });
return result;

效果怎么样？ API 调用从 43,200 次/月 降到约 30 次/月（每分钟 1 次变每天 1 次），节省 99.9%。

踩坑记录

改完发现插件冲突了——系统自带一个 feishu，我自己又装了一个。解决方式：

# 删除手动版，保留系统版
rm -rf /root/.openclaw/extensions/feishu

铁律：修改配置前一定先备份，重启服务用安全脚本（后面会讲）。

第二步：每日简报脚本，让 AI 主动汇报

等 AI 汇报不如让它主动说。咱们写个 daily-briefing.sh，每天早上 8 点自动推送系统状态。

脚本核心逻辑 

#!/bin/bash
# daily-briefing.sh - 每日系统简报

# 1. 收集系统负载
LOAD=$(uptime | awk -F'load average:' '{print $2}' | cut -d',' -f1 | tr -d ' ')

# 2. 磁盘使用率
DISK=$(df -h / | awk 'NR==2 {print $5 " used (" $3 "/" $2 ")"}')

# 3. 内存使用（修复空值问题）
MEM_INFO=$(free -h | awk 'NR==2 {printf "%s / %s (%.0f%% 已用)", $3, $2, ($3/$2)*100}')

# 4. OpenClaw 版本
OC_VERSION=$(openclaw --version 2>/dev/null || echo "未知")

# 5. 昨日清理统计
SESSIONS_CLEANED=$(find /root/.openclaw/sessions -type d -mtime +2 | wc -l)

# 6. 组装消息
msg="🦉 *OpenClaw 每日简报*
━━━━━━━━━━━━━━━━
📅 日期：$(date +%Y-%m-%d)
🖥️ 系统负载：$LOAD
💾 磁盘：$DISK
🧠 内存：$MEM_INFO
📦 版本：$OC_VERSION
🧹 清理会话：$SESSIONS_CLEANED 个
━━━━━━━━━━━━━━━━
一切正常，继续搬砖！"

# 7. 发送到飞书（注意用个人 OpenID，不是群聊 ID！）
openclaw message send \
  --channel feishu \
  --target "ou_b4f317fba4dd5f102eeb6b4e91bd45fb" \
  --message "$msg"

关键修复点

  1. 内存显示为空：一开始 free -h 解析出来是空的，后来优化了 awk 逻辑，现在显示成 718Mi / 5.0Gi (14% 已用)。
  2. 飞书 ID 用错：别用 oc_ 开头的群聊 ID，要用 ou_ 开头的个人 OpenID，不然消息全发群里了。
  3. Cron 格式错误：添加定时任务时，空行会导致解析失败。正确姿势：
# 每天 08:00 执行
cat <<EOF | crontab -
0 8 * * * /root/.openclaw/workspace/scripts/daily-briefing.sh
EOF

第三步：SOSO 菜单，一键直达指挥中心

命令太多记不住？搞个交互式菜单。

创建 soso.sh：

#!/bin/bash
# soso.sh - OpenClaw 快捷菜单
echo "🦉 OpenClaw 指挥中心"
echo "━━━━━━━━━━━━━━━━━━━━"
select option in "启动服务" "重启服务" "查看日志" "升级系统" "清理会话" "查看状态" "退出"; do
  case $option in
    "启动服务") openclaw gateway start ;;
    "重启服务") /root/.openclaw/claw-safe-start.sh ;;
    "查看日志") openclaw logs --tail 50 ;;
    "升级系统") /root/.openclaw/workspace/scripts/upgrade-openclaw.sh --yes ;;
    "清理会话") /root/.openclaw/workspace/scripts/cleanup-sessions.sh ;;
    "查看状态") openclaw gateway status ;;
    "退出") break ;;
  esac
done

加个别名到 ~/.bashrc：

alias soso='/root/.openclaw/workspace/scripts/soso.sh'

以后在终端输入 soso，直接呼出菜单，妈妈再也不用担心我记不住命令了。

第四步：安全机制，改配置前请先请示

这是用血泪换来的教训。

有次我直接修改了 openclaw.json 然后重启服务，差点把配置搞崩。铁律来了：

任何修改系统、配置、重启服务的操作，必须先请示用户，然后用安全脚本执行。

安全启动脚本 claw-safe-start.sh 做了什么？

# 1. 备份当前配置（带时间戳）
cp openclaw.json openclaw.json.backup.$(date +%Y%m%d_%H%M%S)
# 2. 尝试重启
openclaw gateway restart
# 3. 验证启动是否成功
if ! openclaw gateway status; then
  # 4. 失败自动回滚
  echo "启动失败，回滚配置..."
  cp openclaw.json.backup.latest openclaw.json
fi

记住：升级、重启、改配置，100% 走安全脚本，带回滚保障。

成果展示：现在的日常有多爽？

折腾完这一套，现在的日常操作：

  1. 每天早上 8 点：飞书准时收到系统简报，负载、内存、磁盘一目了然
  2. 打开终端：输入 soso 呼出菜单，启动/重启/日志一键搞定
  3. 升级系统：soso 菜单选升级，自动备份 + 升级 + 验证，全程不用管
  4. API 额度：从 43,200 次/月 降到 30 次/月，随便造

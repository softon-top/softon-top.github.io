一、什么是Aria2？

Aria2是一款轻量级、跨平台的命令行下载工具，支持HTTP/HTTPS、FTP、SFTP、BitTorrent、Metalink等多种协议，凭借超高的下载速度、极低的资源占用和灵活的配置能力，成为众多用户心中的「下载神器」。

但Aria2本身无图形界面，搭配AriaNg（一款基于网页的Aria2前端管理工具），可实现可视化操作，兼顾命令行的高效与图形界面的易用性。

二、准备工作：下载所需文件

1. 下载Aria2

  • 下载地址：aria2官方发布页
  • 选择版本：aria2-1.37.0-win-64bit-build1.zip（Windows 64位）

2. 下载AriaNg

  • 下载地址：AriaNg官方发布页
  • 选择版本：AriaNg-1.3.13-AllInOne.zip（全能版，无需部署服务器）

3. 下载Servy（用于将Aria2注册为系统服务）

  • 下载地址：Servy官方仓库
  • 选择版本：servy-6.4-x64-installer.exe

4. 下载Chrome扩展（可选，快速调用Aria2）

  • 扩展地址：Aria2 整合 - Chrome应用商店

三、安装与文件解压

1. 创建核心目录

在D盘创建主目录：D:\Program Files\Aria2
再创建AriaNg子目录：D:\Program Files\Aria2\AriaNg

2. 解压文件

  • 将aria2-1.37.0-win-64bit-build1.zip解压到D:\Program Files\Aria2（解压后确保目录内有aria2c.exe）；

  • 将AriaNg-1.3.13-AllInOne.zip解压到D:\Program Files\Aria2\AriaNg（解压后可见index.html）；

双击D:\Program Files\Aria2\AriaNg\index.html，自动在浏览器打开AriaNg界面

但是现在打开会提示出错，我们继续操作

四、配置Aria2（核心步骤）

在D:\Program Files\Aria2目录下新建aria2.conf文件，右键用记事本打开，填写以下配置内容（根据需求可自行调整）： 

# RPC 设置（AriaNg连接核心）
enable-rpc=true
rpc-listen-all=true
rpc-allow-origin-all=true
rpc-listen-port=6800
rpc-secret=123456

# 下载设置
dir=D:\Downloads  # 下载文件保存路径
continue=true     # 支持断点续传
max-concurrent-downloads=5  # 最大同时下载任务数
max-connection-per-server=16  # 单服务器最大连接数
split=16  # 分块下载数
min-split-size=10M  # 最小分块大小

# 日志设置
log-level=warn  # 日志级别（warn仅记录警告，减少日志体积）
console-log-level=error  # 控制台日志级别

# BT 设置（可选，优化BT下载）
bt-enable-lpd=true
bt-max-peers=50

注意：D:\Downloads目录需提前创建，或者你可以自定义自己的下载文件夹。

五、临时启动Aria2（测试基础功能）

打开CMD命令提示符，依次执行以下命令：

1. cd /d "D:\Program Files\Aria2"
2. aria2c.exe --conf-path=aria2.conf

若命令行无报错，说明Aria2基础配置正常，再打开提示已连接表示成功。

点击Aria2状态显示如下

六、安装Chrome扩展（便捷调用）

  1. 打开Chrome扩展商店，安装「Aria2 整合」扩展；

  2. 扩展设置中填写Aria2的RPC地址和密钥（与AriaNg一致，rpc-secret=123456，可自定义修改）；

  3. 浏览网页时，右键点击下载链接，选择「使用Aria2下载」，即可快速添加任务；

七、注册Aria2为系统服务（开机自启）

1. 直接安装Servy

默认安装到C:\Program Files\Servy，可以自定义文件夹，安装后在开始菜单可见。

2. 创建服务配置

直接点击Servy创建Aria2服务，填好后点击Install安装成服务

3. 启动服务

点击Servy Manager 选择创建的aria2服务 点启动

八、常见问题排查

  1. AriaNg连接失败：

    • 检查Aria2是否启动；
    • 核对aria2.conf中RPC端口、密钥是否与AriaNg一致；
    • 关闭防火墙/杀毒软件，避免拦截6800端口。

  2. 下载速度慢：

    • 调整max-connection-per-server、split参数（数值越大，分块越多，速度可能越快）；
    • BT下载可添加公共Tracker地址，提升节点数。

  3. 服务注册失败：

    • 确保Servy填写的路径及参数是否正确。

九、最终效果展示

通过以上步骤，即可实现：

  • Aria2后台运行+AriaNg可视化管理，支持多协议下载、断点续传；
  • Chrome扩展一键调用，无需手动复制链接；
  • 注册系统服务后开机自启，无需每次手动启动CMD。

十、总结

  1. 先完成Aria2基础配置和临时启动，验证核心功能可用；
  2. 再配置AriaNg可视化和Chrome扩展，提升使用体验；
  3. 最后注册系统服务，实现开机自启，完成完整的下载体系搭建。

Aria2+AriaNg的组合兼顾速度与易用性，是Windows系统下高效的下载解决方案。
